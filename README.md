## 📄 컨벤션 및 브랜치 전략

<h3>Commit Convention</h3>

```
- init: 초기 세팅
- feat: 새로운 기능 추가 (새로운 구현) 
- fix: 버그 수정 
- docs: 문서 추가, 수정, 삭제 
- style: UI 디자인 변경  
- refactor: 코드 리팩토링 
- chore: 그 외 자잘한 수정, 코드 포맷팅(코드 순서 변경, 세미콜론 추가) (주석 삭제)
```

<h3>Issue / Pull Request Convention</h3>

```
[종류] 작업 이름
ex) [FEAT] 회원가입 구현 및 이메일 본인인증, 비밀번호 찾기
```
`담당자`, `라벨` 추가하기
